# PLC-MCS 장애 조치 SOP

## 문서 목적

이 문서는 PLC 이벤트 오류, 필드 누락, 인터락, 위치 불일치가 발생했을 때 운영자가 확인해야 할 순서와 조치 기준을 정의합니다.
AI 운영 챗봇은 이 문서를 근거로 장애 원인과 다음 확인 작업을 설명합니다.

## 공통 조치 원칙

- AI는 설비나 이동오더를 직접 조작하지 않습니다.
- AI는 원인 후보와 확인 순서를 안내합니다.
- 실제 복구 조치는 운영자 또는 설비 담당자가 화면과 현장을 확인한 뒤 수행합니다.
- 필수 필드 누락은 PLC 송신 payload 생성부와 태그 매핑표를 우선 확인합니다.
- 설비 오류와 인터락은 안전 조건을 먼저 확인한 뒤 재시작 여부를 판단합니다.

## 장애 분류

| 장애 유형 | 대표 이벤트 | 주요 증상 | 우선 확인 위치 |
|---|---|---|---|
| 필드 누락 | `VALIDATION_FAILED` | MCS가 이벤트를 처리하지 못함 | PLC payload 생성부, 태그 매핑표 |
| 설비 오류 | `EQUIPMENT_ERROR` | 이동오더 실패, 설비 정지 | PLC 알람 코드, 설비 현장 상태 |
| 인터락 | `INTERLOCK_BLOCKED` | 이동 시작 또는 진행 차단 | 안전문, 센서, 선행 조건 |
| 위치 불일치 | `ARRIVED_WRONG_LOCATION` | 자재가 예상 도착지와 다른 위치에서 감지 | 위치 센서, 경로 설정, 목적지 태그 |
| LOT 불일치 | `TRANSFER_STARTED`, `TRANSFER_COMPLETED` | 재고 반영 또는 이동 추적 불가 | LOT 인식 장치, `LOT_NO` 태그 |

## SOP-001. TRANSFER_STARTED 필드 누락

### 증상

- MCS PLC 이벤트에 `PROCESS_RESULT = VALIDATION_FAILED`가 기록됩니다.
- 이동오더가 `IN_PROGRESS`로 전환되지 않거나, 이동 시작 근거가 부족합니다.
- AI 분석에서 `toLocationCd`, `lotNo`, `targetId` 누락이 언급됩니다.

### 원인 후보

| 누락 필드 | 원인 후보 | 확인 태그 |
|---|---|---|
| `toLocationCd` | 도착지 태그가 payload에 매핑되지 않음 | `TO_LOCATION_CD` |
| `lotNo` | LOT 태그가 payload에 매핑되지 않음 | `LOT_NO` |
| `targetId` | MCS 이동오더 ID가 PLC로 전달되지 않음 | `TRANSFER_ID` |
| `equipmentCd` | 설비 코드 상수 또는 공통 블록 초기화 누락 | `EQUIPMENT_CD` |

### 확인 순서

1. MCS의 PLC 이벤트 화면에서 `PROCESS_MESSAGE`를 확인합니다.
2. 누락 필드가 무엇인지 확인합니다.
3. PLC 태그 매핑표에서 누락 필드에 대응하는 태그를 찾습니다.
4. PLC 송신 payload 생성부에서 해당 태그가 JSON 필드에 대입되는지 확인합니다.
5. 태그 값이 비어 있으면 이동 명령 수신부 또는 센서 판정부를 확인합니다.
6. 수정 후 동일 이동 시작 이벤트를 재전송하거나 새 이동오더로 재시도합니다.

### AI 답변 예시

```text
이동 시작 이벤트에서 도착 위치나 LOT 정보가 빠진 상태예요.
MCS는 이동할 목적지와 대상 LOT를 알아야 이동 상태를 확정할 수 있습니다.
PLC 송신부에서 TO_LOCATION_CD와 LOT_NO가 payload에 들어가는지 먼저 확인해보세요.
```

## SOP-002. TRANSFER_COMPLETED 필드 누락

### 증상

- 이동 완료 이벤트는 들어왔지만 재고 반영이 되지 않습니다.
- 완료 위치 또는 LOT가 확인되지 않아 MCS가 완료 처리를 보류합니다.

### 원인 후보

| 누락 필드 | 원인 후보 | 확인 태그 |
|---|---|---|
| `locationCd` | 도착 위치 센서 값이 payload에 매핑되지 않음 | `CURRENT_LOCATION_CD` |
| `lotNo` | LOT 인식 결과가 payload에 매핑되지 않음 | `LOT_NO` |
| `targetId` | 완료 대상 이동오더 ID가 누락됨 | `TRANSFER_ID` |

### 확인 순서

1. 완료 이벤트의 `targetId`가 실제 이동오더와 일치하는지 확인합니다.
2. `locationCd`가 실제 도착 Location과 일치하는지 확인합니다.
3. `lotNo`가 이동 대상 LOT와 일치하는지 확인합니다.
4. 재고 반영이 되지 않았다면 MCS 이동 관리 화면에서 이동 상태를 확인합니다.
5. 필요 시 현장 LOT와 시스템 LOT를 대조합니다.

## SOP-003. EQUIPMENT_ERROR 설비 오류

### 증상

- PLC 이벤트 타입이 `EQUIPMENT_ERROR`로 들어옵니다.
- 관련 이동오더가 `FAILED` 상태가 됩니다.
- 알림 또는 AI 브리핑에서 설비 오류가 표시됩니다.

### 원인 후보

| 오류 코드 | 의미 | 우선 확인 |
|---|---|---|
| `MOTOR_OVERLOAD` | 모터 과부하 | 모터 부하, 컨베이어 걸림, 적재 중량 |
| `SENSOR_FAIL` | 센서 이상 | 감지 센서, 배선, 센서 전원 |
| `EMERGENCY_STOP` | 비상정지 | 비상정지 버튼, 안전 릴레이 |
| `PLC_COMM_ERROR` | PLC 통신 오류 | 네트워크, PLC 통신 모듈, MCS 수신 로그 |

### 확인 순서

1. 오류 코드와 메시지를 확인합니다.
2. 설비 현장 상태와 PLC 알람 화면을 확인합니다.
3. 이동 중인 자재가 있는 경우 안전 상태를 먼저 확인합니다.
4. MCS 이동 관리 화면에서 실패 처리된 이동오더를 확인합니다.
5. 원인 해소 후 이동오더 재생성 또는 재시작 절차를 수행합니다.

## SOP-004. INTERLOCK_BLOCKED 인터락 차단

### 증상

- 이동 시작 또는 진행 중 `INTERLOCK_BLOCKED` 이벤트가 발생합니다.
- 설비가 안전 조건 미충족으로 이동을 차단합니다.

### 원인 후보

| 원인 | 확인 위치 |
|---|---|
| 안전문 열림 | Safety Door 센서 |
| 선행 공정 미완료 | 상위 공정 완료 신호 |
| 목적지 점유 | 도착 Location 재고/센서 |
| 비상정지 상태 | E-Stop 회로 |
| 작업 모드 불일치 | 자동/수동 모드 스위치 |

### 확인 순서

1. 인터락 메시지와 오류 코드를 확인합니다.
2. 안전문, 비상정지, 자동/수동 모드 상태를 확인합니다.
3. 목적지 Location이 비어 있는지 확인합니다.
4. 선행 공정 완료 신호가 들어왔는지 확인합니다.
5. 조건이 정상화되면 PLC 인터락 해제 후 이동 재시도를 검토합니다.

## SOP-005. ARRIVED_WRONG_LOCATION 위치 불일치

### 증상

- 자재가 예상 도착 위치와 다른 위치에서 감지됩니다.
- MCS가 관련 이동오더를 실패 처리합니다.

### 원인 후보

| 원인 | 확인 위치 |
|---|---|
| 목적지 태그 오류 | `TO_LOCATION_CD` |
| 현재 위치 센서 오류 | `CURRENT_LOCATION_CD` |
| 경로 설정 오류 | MCS 경로 관리/경로 최적화 |
| 수동 이동 개입 | 현장 이동 이력 |

### 확인 순서

1. 이벤트의 `locationCd`와 이동오더의 `toLocationCd`를 비교합니다.
2. MCS 경로 관리 화면에서 경로가 올바른지 확인합니다.
3. PLC의 현재 위치 센서 값이 실제 위치와 일치하는지 확인합니다.
4. 수동 이동 또는 작업자 개입이 있었는지 확인합니다.
5. 실제 위치와 시스템 위치를 정합화한 뒤 이동오더를 재처리합니다.

## AI 답변 가이드

AI는 장애 조치 답변에서 다음 표현을 우선 사용합니다.

| 상황 | 권장 표현 |
|---|---|
| 필드 누락 | "필수 데이터가 빠져서 MCS가 이동 상태를 확정하지 못했습니다." |
| 태그 매핑 확인 | "PLC 송신부에서 해당 태그가 payload 필드로 들어가는지 확인하세요." |
| 설비 오류 | "설비 알람 코드와 현장 상태를 먼저 확인하세요." |
| 인터락 | "안전 조건과 목적지 점유 상태를 먼저 확인하세요." |
| 위치 불일치 | "현재 감지 위치와 이동오더 목적지를 대조하세요." |

AI는 다음 표현을 피합니다.

- "제가 이동오더를 재시작하겠습니다."
- "제가 재고를 보정하겠습니다."
- "PLC 코드를 자동 수정했습니다."
- 근거 없이 특정 원인을 단정하는 표현
